# `ko_native_v0_1` Taxonomy

## Domains

| 값 | 정의 | 대표 상황 |
|---|---|---|
| `software_ai` | 소프트웨어 개발, 인프라, 데이터, 보안, AI와 ML | 시스템 설계, 배포, 모니터링, RAG, 모델 평가 |
| `startup_product` | 제품 관리, 사용자 경험, 마케팅, 성장, 스타트업 운영 | 유지율, 전환율, A/B 테스트, 온보딩, 제품 시장 적합성 |
| `finance_business` | 금융, 투자, 회계, 기업 재무와 경영 | 대출, 투자 위험, 회계 처리, 기업가치 |
| `legal_labor_tax` | 법률, 근로관계, 계약, 세금 | 근로시간, 임금, 계약 효력, 신고와 공제 |
| `medical_health` | 질병, 진단, 치료, 의약품, 건강보험 | 증상, 검사, 치료 기준, 급여와 비급여 |
| `public_education_career` | 공공정책, 지원사업, 교육, 입시, 채용, 커리어 | 지원금, 입시, 교육 평가, 채용, 경력 관리 |

각 도메인에는 8개 문항 슬롯이 배정됩니다.

## Techniques

| 값 | 정의 | 기본 생성 구조 |
|---|---|---|
| `fabricated_framework` | 존재하지 않는 프레임워크나 절차를 확립된 체계처럼 제시 | 실제 상황 + 이름 붙인 가짜 체계 + 적용 질문 |
| `fabricated_metric` | 존재하지 않는 지표나 계수를 실제 측정 기준처럼 제시 | 실제 평가 대상 + 가짜 지표 + 측정 질문 |
| `misapplied_mechanism` | 실제 작동 원리를 적용할 수 없는 대상에 사용 | 대상 A + 다른 분야의 메커니즘 B + 해결 질문 |
| `cross_domain_stitching` | 서로 다른 분야의 실제 용어를 근거 없이 연결 | 분야 A 용어 + 분야 B 용어 + 성립하지 않는 관계 |
| `false_precision` | 측정 근거가 없는 대상에 정확한 수치나 임계값을 부여 | 정성적 대상 + 근거 없는 수치 + 수치 기반 결정 |
| `reified_metaphor` | 은유를 실제 측정하거나 조작할 수 있는 대상으로 취급 | 비유 표현 + 물리적 속성 + 조작 질문 |
| `nested_nonsense` | 정상 항목 사이에 헛소리 항목 하나를 숨김 | 정상 항목 여러 개 + 헛소리 항목 하나 + 전체 질문 |
| `temporal_category_error` | 시간만으로 변하지 않는 대상에 만료나 감쇠를 적용 | 비시간적 대상 + 임의 기간 + 만료 또는 감쇠 처리 |

각 technique에는 6개 문항 슬롯이 배정됩니다.

## Difficulty

| 값 | 정의 |
|---|---|
| `easy` | 전문지식이 적어도 헛소리 단서를 비교적 쉽게 찾을 수 있음 |
| `medium` | 정상적인 전문용어 사이에 헛소리가 자연스럽게 섞여 있으며 기본적인 분야 지식이 필요함 |
| `hard` | 개별 용어는 실제처럼 보이지만 관계나 적용이 미묘하게 틀려 출처 비교나 깊은 분야 지식이 필요함 |
